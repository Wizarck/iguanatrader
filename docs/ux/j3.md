# Journey 3 — Research briefings + weekly review (DRAFT v1)

> Source: PRD §User Journeys / Journey 3 (weekly review domingo mañana — representative of the Tier-3 routine family: premarket / midday / postmarket / weekly); FR43 (scheduled orchestration routines), FR44 (weekly review PDF), FR58 (methodology profile), FR71 (LLM-synthesised brief with citations + audit_trail), FR72 (refresh on schedule + on-trigger), FR73 (immutable per version), FR74 (proposal-to-brief linkage), FR75 (tier-based feature availability), FR68-FR70 (bitemporal storage + provenance + audit_trail), FR80-FR81 (Hindsight integration write-on + recall toggle). Personas: JTBD-3 (audit + reflection compounding), JTBD-4 (anti-hallucination — every numeric assertion cited, every calculation shows its work).

## 1. Goal

Enable Arturo to (a) consume scheduled routine outputs (premarket briefing, midday check, postmarket summary, weekly review) via the messaging channels he is using AND on the dashboard, (b) drill into any briefing on the dashboard to see the underlying citations + bitemporal facts + show-your-work audit trail, and (c) read the weekly review PDF as the durable artefact that compounds lessons over years.

The journey enforces JTBD-4's anti-hallucination guarantee: **every numeric claim in every brief is one click away from its source fact + retrieval method + provenance metadata**. The system cannot invent values; if a citation breaks, render fails (FR75 spirit).

## 2. Trigger

- **Tier-3 scheduled routines** (FR43, FR35): premarket 06:30 ET, midday 12:30 ET, postmarket 16:30 ET, weekly Sunday 18:00 ET (or per cron config).
- **On-trigger refresh** (FR72): a material new fact (earnings release, 8-K, breaking news with high relevance score) triggers an immediate `research_brief` refresh for the affected symbol → user notified via Tier-1 / Tier-2 channel as appropriate.
- **Manual refresh from dashboard**: from `/research/[symbol]`, Arturo clicks "Refresh brief". Progress streams; new immutable version created (FR73). Rate-limited to 5/min (slowapi).
- **Weekly review PDF**: Sunday 18:00 ET, generated by O2 routine, attached to channel message + linked from `/` home banner for the next 7 days.

## 6. Edge cases

- **Brief synthesis takes >30s** → cancel button appears in `BriefHeader` (per Architecture Decisions §Frontend Architecture loading-states rule). Cancel POSTs to abort; partial brief discarded.
- **LLM cost would exceed daily budget** (FR41) → refresh button shows tooltip "Today's LLM budget exhausted — refresh queued for next budget window" + cancel option. Server enforces — UI just communicates.
- **Source unreachable at synthesis time** (e.g. SEC EDGAR 503) → brief still synthesises with the most recent available facts; `BriefHeader` shows `Badge variant="warn"` "Some sources stale — see FactTimeline for freshness".
- **Tier-B/C feature has no data at as-of timestamp** (FR75) → brief omits that pillar with explicit "no data available at this knowledge-time" copy. Strategies handle `None` returns; UI mirrors that explicitly.
- **Bitemporal as-of in the future** → 422 + error copy "Cannot replay knowledge-time in the future".
- **Brief version dropdown** (when versions accumulate) — collapse to last 5 + "View all versions" affordance.
- **Hindsight recall toggle when brief is mid-refresh** → toggle disabled with tooltip "Refresh in progress — toggle after completion to avoid mid-flight state changes".
- **Weekly PDF generation fails** (plotly OOM, jinja template error) → routine emits a degraded text-only digest via channels + dashboard banner; PDF retry queued; admin sees the failure entry on `/risk` or `/costs` (TBD: where do orchestration failures surface?).

## 7. Decisions specific to J3

- **Citations are first-class atoms** (CitationLink) used in three contexts: brief body inline, FactTimeline rows, AuditTrailViewer input rows. Same component, same visual language — reinforces that "everywhere you see a number, you can click to its source".
- **Audit-trail is its own route**, not a modal. Rationale: deep-link with `#metric=` anchor must work; URL is shareable for retros; PDF export of an audit-trail page is conceivable later.
- **Hindsight presence is a badge, not a tab**. The user knows from the badge whether recall is on; the audit-trail surface (Step 3) shows whether a particular brief version used recall (Hindsight inputs surface as `CitationLink` chips with `method='llm'` + a Hindsight tag).
- **Manual refresh is open to the tenant user**, rate-limited via slowapi 5/min. Per [personas-jtbd.md](../personas-jtbd.md) §RBAC Matrix (refined 2026-05-05), MVP/v2 is single-seat-per-tenant — there is no read-only role to gate against. v3 multi-seat may reintroduce a read-only fallback header showing "Next scheduled refresh: <ISO 8601 ts>".
- **As-of replay deep-link from `/trades`** is the bridge that ties J3 to J1 — the bitemporal contract makes "what did we know on trade day" a single click from the trade history.
- **PDF is a durable artefact**, not a virtual view. Stored under `apps/api/.../reports/weekly/` with retention per Architecture Decisions §Backup strategy (litestream + filesystem). FR50 export is for trade rows; weekly PDFs are kept indefinitely.
- **Weekly PDF lessons section is a candidate list**, not auto-applied. Arturo curates into his own `memoria.md` — preserves human authorship of the journal.

## 8. Notes for implementation

- **Slice R5 (`research-brief-synthesis`)** is the heaviest slice — owns:
  - 5 components: `BriefHeader`, `FactTimeline`, `CitationLink`, `AuditTrailViewer`, `MethodologyBadge`.
  - Routes: `/research/+page.svelte` (watchlist), `/research/[symbol]/+page.svelte` (brief detail), `/research/[symbol]/audit-trail/[brief_version]/+page.svelte` (audit).
  - SSE: `research.brief.refresh.progress`, `research.brief.refreshed`, `research.fact.recorded`.
  - Dev mocks for R2/R3/R4 sources to avoid blocking.
- **Slice O2 (`orchestration-scheduler-routines`)** owns:
  - The 4 cron-scheduled routines (premarket / midday / postmarket / weekly_review).
  - The PDF generator (plotly + jinja per PRD).
  - The dashboard banner SSE event for the routine digest.
- **Slice R6 (`hindsight-integration`)** owns:
  - `/settings` page (toggle UI for `hindsight_recall_enabled`).
  - The Hindsight write-on (FR80) is always-on backend; not user-visible directly except via PDF appendix.
  - The recall toggle's audit trail entry (FR47).
- **Brief body markdown rendering** must use a sanitised renderer (e.g. `marked` + DOMPurify or `markdown-it` with whitelist) — LLM output is untrusted. Custom token handlers map `[fact:<id>]` → `CitationLink` Svelte component.
- **Fact-id citation resolver**: server returns the brief body with `[fact:<id>]` markers AND a fact bundle keyed by id. Client renderer resolves at render time. Broken id → renders as `Badge variant="warn"` "missing fact" + reports to OTel for monitoring.
- **PDF link signing**: each weekly PDF URL is HMAC-signed for tenant + 7-day expiry; longer access via authenticated `/reports` listing page.
- **A11y**: brief body uses semantic headings (`<h2>` per pillar, `<h3>` per sub-section); CitationLink chips have `aria-label="Citation: <sourceLabel> retrieved <retrievedAt>"`; AuditTrailViewer entries are accessible accordion (one open at a time, keyboard navigable).
- **Tokens**: FactTimeline tier badges + MethodologyBadge colour map are TBD in DESIGN.md (per [components.md](components.md#0-conventions) §0.3). Sally to lock during the variant round.
- **Performance**: `/research/[symbol]` page has the densest layout in the dashboard — Lighthouse target ≥90 perf may need EquityCurve-style virtualisation if facts > 200. SkeletonLoader for above-fold blocks while audit_trail / facts load async.
