## 1. Goal

Enable Arturo to (a) consume scheduled routine outputs (premarket briefing, midday check, postmarket summary, weekly review) via the messaging channels he is using AND on the dashboard, (b) drill into any briefing on the dashboard to see the underlying citations + bitemporal facts + show-your-work audit trail, and (c) read the weekly review PDF as the durable artefact that compounds lessons over years.

The journey enforces JTBD-4's anti-hallucination guarantee: **every numeric claim in every brief is one click away from its source fact + retrieval method + provenance metadata**. The system cannot invent values; if a citation breaks, render fails (FR75 spirit).

## 2. Trigger

- **Tier-3 scheduled routines** (FR43, FR35): premarket 06:30 ET, midday 12:30 ET, postmarket 16:30 ET, weekly Sunday 18:00 ET (or per cron config).
- **On-trigger refresh** (FR72): a material new fact (earnings release, 8-K, breaking news with high relevance score) triggers an immediate `research_brief` refresh for the affected symbol â†’ user notified via Tier-1 / Tier-2 channel as appropriate.
- **Manual refresh from dashboard**: from `/research/[symbol]`, Arturo clicks "Refresh brief" (admin only). Progress streams; new immutable version created (FR73).
- **Weekly review PDF**: Sunday 18:00 ET, generated by O2 routine, attached to channel message + linked from `/` home banner for the next 7 days.

## 6. Edge cases

- **Brief synthesis takes >30s** â†’ cancel button appears in `BriefHeader` (per Architecture Decisions Â§Frontend Architecture loading-states rule). Cancel POSTs to abort; partial brief discarded.
- **LLM cost would exceed daily budget** (FR41) â†’ refresh button shows tooltip "Today's LLM budget exhausted â€” refresh queued for next budget window" + cancel option. Server enforces â€” UI just communicates.
- **Source unreachable at synthesis time** (e.g. SEC EDGAR 503) â†’ brief still synthesises with the most recent available facts; `BriefHeader` shows `Badge variant="warn"` "Some sources stale â€” see FactTimeline for freshness".
- **Tier-B/C feature has no data at as-of timestamp** (FR75) â†’ brief omits that pillar with explicit "no data available at this knowledge-time" copy. Strategies handle `None` returns; UI mirrors that explicitly.
- **Bitemporal as-of in the future** â†’ 422 + error copy "Cannot replay knowledge-time in the future".
- **Brief version dropdown** (when versions accumulate) â€” collapse to last 5 + "View all versions" affordance.
- **Hindsight recall toggle when brief is mid-refresh** â†’ toggle disabled with tooltip "Refresh in progress â€” toggle after completion to avoid mid-flight state changes".
- **Weekly PDF generation fails** (plotly OOM, jinja template error) â†’ routine emits a degraded text-only digest via channels + dashboard banner; PDF retry queued; admin sees the failure entry on `/risk` or `/costs` (TBD: where do orchestration failures surface?).

## 7. Decisions specific to J3

- **Citations are first-class atoms** (CitationLink) used in three contexts: brief body inline, FactTimeline rows, AuditTrailViewer input rows. Same component, same visual language â€” reinforces that "everywhere you see a number, you can click to its source".
- **Audit-trail is its own route**, not a modal. Rationale: deep-link with `#metric=` anchor must work; URL is shareable for retros; PDF export of an audit-trail page is conceivable later.
- **Hindsight presence is a badge, not a tab**. The user knows from the badge whether recall is on; the audit-trail surface (Step 3) shows whether a particular brief version used recall (Hindsight inputs surface as `CitationLink` chips with `method='llm'` + a Hindsight tag).
- **Manual refresh is admin-only** (FR4 + FR81 align). Non-admin sees a read-only header with "Next scheduled refresh: 2026-05-01 12:30 UTC".
- **As-of replay deep-link from `/trades`** is the bridge that ties J3 to J1 â€” the bitemporal contract makes "what did we know on trade day" a single click from the trade history.
- **PDF is a durable artefact**, not a virtual view. Stored under `apps/api/.../reports/weekly/` with retention per Architecture Decisions Â§Backup strategy (litestream + filesystem). FR50 export is for trade rows; weekly PDFs are kept indefinitely.
- **Weekly PDF lessons section is a candidate list**, not auto-applied. Arturo curates into his own `memoria.md` â€” preserves human authorship of the journal.
