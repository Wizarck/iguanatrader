# Journey 1 — Authenticated user portfolio surface (DRAFT v1)

> Source: PRD §User Journeys / Journey 1 (happy path, narrated as Telegram approval but the dashboard is the persistent surface throughout); FR31, FR38 (auth + authorized-senders enforcement), FR46-FR49 (persistence), FR54 (dashboard surface), FR55 (kill-switch button), FR12-FR18 (trade lifecycle visible in dashboard), FR1-FR5 (strategy management surface). Personas: JTBD-1 (hands-off pipeline + veto power), JTBD-2 (programmatic risk policy), JTBD-6 (trustworthy unattended). Architecture Decisions: cookie-based JWT session (HttpOnly + Secure + SameSite=Strict), 7-day sliding lifetime, SvelteKit `(app)` route group authenticated-only.

## 1. Goal

Enable Arturo to authenticate once via the web login surface, then live inside the authenticated dashboard as the **persistent operational surface** — observing portfolio state, equity curve, open positions, trade history, and active strategies — while the autonomous proposal pipeline runs in the background and surfaces its outputs through Journey 2 (approval flow) and Journey 3 (research briefings + weekly review).

This journey covers everything the user needs **between** approval moments: the read-mostly surface that makes Arturo trust the system enough to step away (JTBD-6).

## 2. Trigger

- **Cold start**: Arturo opens `https://localhost:<port>/` from his Windows host. JWT cookie missing or expired → redirected to `/login`.
- **Warm session**: JWT cookie valid (≤7 days since last activity) → redirected to `/` dashboard.
- **Tab return**: SvelteKit hydrated tab regains focus → `riskStore` + `equityStore` reconnect to SSE; stale-data banner if reconnection takes >2s.

## 6. Edge cases

- **Approval arrives mid-Journey-1** → see [j2.md](j2.md). The compact `ApprovalCard` preview on the home page is the bridge.
- **Research drilldown from a trade row** → see [j3.md](j3.md) Step 2 and Step 3.
- **TWS Gateway disconnect** → mirrored from PRD Journey 4 (failure recovery). The `ConnectionIndicator` + RiskCapsBar surface convey state; the Telegram/WhatsApp Tier-1 alert is the proactive channel — not duplicated as a toast in the dashboard.
- **Multi-tab open**: `tenantStore` + JWT cookie shared; SSE connections per-tab. Each tab can run independent filters/views; mutating actions (approve, override, force-exit) propagate via SSE so all tabs converge.
- **Cold-tab reconnect after suspend** (laptop closed): freeze data; show "stale" banner; SSE reconnects with last-event-id to backfill missed events.

## 7. Decisions specific to J1

- **Top-of-page kill-switch is global, not per-page**. Compact KillSwitchButton in the (app) top bar is wired once at the layout level (per [components.md](components.md#layout--navigation)). Rationale: every screen must allow halting the system in 1 action (FR55 + JTBD-6 trust).
- **Login is its own route group `(auth)/`** — keeps the (app) authenticated layout clean; no Sidebar on login.
- **Active-route prefix matching in Sidebar**: deep routes like `/research/AAPL/audit-trail/3` keep "Research" highlighted (decision rationale in [components.md](components.md#22-sidebarsvelte) Sidebar Behaviour).
- **No PWA in MVP**: per Architecture Decisions §Frontend Architecture, `vite-plugin-pwa` configured but disabled until v2 hosts dashboard outside localhost.
- **Theme toggle in Sidebar footer** (not user menu) — light/dark + system preference per Architecture Decisions §Frontend Architecture.
- **Date format ISO 8601 absolute + relative pair** everywhere (per memory `feedback_date_format_preference.md`). No relative-only ("3h ago") without the absolute timestamp companion.

## 8. Notes for implementation

- **Slice 4 (`auth-jwt-cookie`)** lands the `(auth)/login/+page.svelte`, the JWT cookie middleware, the redirect-to behaviour, and the form action. It does NOT land the home page content (slice W1 + T4 do that).
- **Slice W1 (`dashboard-svelte-skeleton`)** lands the `(app)/+layout.svelte`, Sidebar dynamic enumeration, `+error.svelte`, ConnectionIndicator, and the route placeholders for portfolio/trades/strategies (returning "loading…" until T4 fills them in). The kill-switch top-bar slot also lands here, even though K1 owns the underlying state — slot wiring is layout-level.
- **Slice T4 (`trading-routes-and-daemon`)** brings the `/portfolio`, `/trades`, `/strategies` routes online with full impl: tables, filters, force-exit, export.
- **SSE event contract** (per Architecture Decisions §API & Communication Patterns) — typed events `event:` + `id:` for resume + `data:` JSON. The reconnect indicator must show last-event-id during reconnect so the user can see catch-up progress.
- **CSV/PDF export** uses `Content-Disposition: attachment` with deterministic filenames `iguanatrader-trades-<tenant>-<from>-<to>.<ext>`.
- **CSP headers** set in `hooks.server.ts`: deny inline scripts (Svelte SSR safe); allow `connect-src` to API and SSE endpoints; allow TradingView Lightweight Charts CDN if used (TBD: bundle vs CDN).
- **A11y catch list (eslint-plugin-svelte)**: every `Input` paired with a real `<label>`; every icon-only `Button` carries `aria-label`; every interactive `Card` is `<button>` semantics; `+error.svelte` uses `role="alert"` for unrecoverable states.
