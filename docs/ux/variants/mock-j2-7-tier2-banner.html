The three options compared

A — Banner only

  • Surface: banner sticky at top + dismissable.
  • Loses history: once dismissed (or auto-dismissed when cap drops), the user has no record of "the system warned me at 14:32".
  • Audit gap: J2 §6 (audit trail) cites risk_overrides but cap warnings aren't override events — they need their own audit surface.
  • Implementation: 1 component (Banner). Cheapest.

B — Timeline only

  • Surface: chronological event list as a section.
  • No urgency cue: a Tier-2 entry sits 4th from the top after newer events; user scanning the page may miss it; no dismissal interaction.
  • History preserved cleanly: every cap event is a row with timestamp.
  • Implementation: 1 component (Timeline list).

C — Banner + Timeline

  • Banner = active state (urgent, dismissable, auto-reappears on next state change).
  • Timeline = history (audit trail of cap events with timestamps).
  • The same event lives in both: banner shows current; timeline shows history. Cross-reference visible (the timeline row that mirrors the banner shows a small "↑ same event" anchor).
  • Same pattern Datadog / PagerDuty use: active state at top, history below. Familiar.
  • Implementation: 2 components (Banner + Timeline). Both already needed elsewhere — Banner is reused on stale-data alerts (J1 §3 step 1), Timeline is reused on broker reconnects.
  • Cost: ~10% more LOC than B alone (Banner is small).

My read

A alone fails the audit / history requirement (cap warnings need to be reviewable later — Arturo will retro "what happened on 2026-05-05?"). B alone fails the urgency requirement (a fresh Tier-2 buried 4th in a list won't trigger action). The "noise" cost of both is small because the banner auto-clears when state drops below threshold; on healthy days the page is just caps + timeline.

→ Recommendation: Option C — Banner + Timeline

Banner sticky at top of /risk while a Tier-2 cap is active (auto-dismisses when state drops below 80% threshold; user "dismiss" hides it for the session but it reappears if a new threshold breach occurs). Timeline section below caps shows chronological history (last 24h on /risk; full history on a future /risk/events route). Banner mirror-anchors to the timeline row via "↑ same event" link. Tier-1 (cap fully consumed) reuses the Banner shape with destructive tokens instead of warn-bg.
