My read

YAML indentation is load-bearing — break it and the strategy fails to parse. A textarea offers ZERO indent assistance; for an 18-line file that's measurable friction every single edit. Monaco gives the same value as CodeMirror 6 but at 8.5× the bundle. CodeMirror 6 is the consensus pick for "I need a code editor in my SaaS app" in 2026 — used by Linear, Sourcegraph (post-Monaco migration), Replit, GitHub. @codemirror/lang-yaml is maintained, ESM-native, theme-ready (the syntax tokens above can be derived from our locked DESIGN.md tokens — accent for keys, success for strings, warn-bg for numbers, destructive for booleans, mute for comments).

→ Recommendation: B — CodeMirror 6 minimal build

Stack: codemirror + @codemirror/lang-yaml + @codemirror/theme-one-dark overridden with our locked OKLCH tokens (custom theme exports a iguanaDarkTheme for the dark ones — light derivation lands in slice W1). Bundle ≈70 KB gzipped, lazy-loaded when the "Edit YAML" modal opens (deferred dynamic import). YAML schema validation: client-side basic via the language module + server-side definitive (always — never trust the client). Errors surface inline (squiggles + status row). Implementation lands in slice T4 inside the strategy edit modal. Ship as a small wrapper component YAMLEditor.svelte (props: value, onChange, readonly, schema) so future YAML surfaces (risk policy editor, methodology config) reuse it.
