The structural decision

Where do these surfaces live in the docs?

Option A — Fold into J1 §0

  • Status quo. j1.md already has Step 0 (login) at lines 37-46.
  • +error.svelte would land as a §0.5 sub-section.
  • Pro: doc count stays at 4. Login is sub-second; not a "journey".
  • Con: J1 doc grows; mixes the gate (login/error) with the destination (dashboard).
  • Con: slice 4 (auth-jwt-cookie) cites j1.md for a surface that's structurally not J1's responsibility.

Option B — Promote to J0 (Auth + global error surfaces)

  • New file docs/ux/j0.md; covers /login, /+error.svelte (404/500/401), and the rate-limit banner edge case.
  • j1.md drops Step 0; gains a 1-line "see J0 for the gate".
  • Pro: slice 4 cites j0.md cleanly; slice W1 (dashboard skeleton) cites j1.md; concerns separated.
  • Pro: future RBAC/2FA/MFA additions for v3 land in J0 without bloating J1.
  • Pro: error surfaces are not "post-login" — they're cross-cutting; pinning them to J1 is a category mistake.
  • Con: doc count: 4 → 5. Marginal.
Notes: all surfaces use locked tokens (DESIGN.md §1) and Lucide icons. Brand mark "i" is a placeholder until logo/iconography lands as a separate decision (out of scope for C3).
