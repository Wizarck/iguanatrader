iguanatrader · Brief citation marker syntax

Decision: J3-2 — how does the LLM mark citations inline in the synthesised brief body?  ·  Date: 2026-05-05  ·  Surface: the brief-body renderer at [j3.md](../j3.md) §3 Step 2 (`/research/[symbol]`)  ·  Sample: 2 sentences citing 4 facts (NVDA fundamentals)

The LLM (research_brief_generator) outputs synthesised prose. Each numeric assertion must cite the fact it relies on. The question: how. Three encodings — same prose, same 4 facts, three syntaxes. The renderer in apps/web/src/lib/research/render-brief.ts parses the marker; the user sees the same rendered chip regardless.

A — [fact:<id>] inline
Raw source from LLM
EPS TTM is $3.18 [fact:f_eps_aapl_q1_2026], beating consensus by $0.18 [fact:f_eps_surprise_q1_2026]. FY estimate revised up to $3.62 [fact:f_fyest_aapl_2026]; analyst price target consensus moved to $920 [fact:f_target_aapl_2026_05].
Rendered (what the user sees)
EPS TTM is $3.18 eps_q1, beating consensus by $0.18 surprise. FY estimate revised up to $3.62 fy_est; analyst price target consensus moved to $920 target.
renderer: 1 regex pass · prompt complexity: low · LLM stability across regen: high
B — [^id] markdown footnote-style
Raw source from LLM
EPS TTM is $3.18[^eps_q1], beating consensus by $0.18[^surprise]. FY estimate revised up to $3.62[^fy_est]; analyst price target consensus moved to $920[^target]. [^eps_q1]: f_eps_aapl_q1_2026 [^surprise]: f_eps_surprise_q1_2026 [^fy_est]: f_fyest_aapl_2026 [^target]: f_target_aapl_2026_05
Rendered native (what GitHub / Pandoc would do)
EPS TTM is $3.18eps_q1, beating consensus by $0.18surprise. FY estimate revised up to $3.62fy_est; analyst price target consensus moved to $920target.

↑ native footnote render = superscript link → scroll-to-bottom. Wrong UX: we want inline chips with fact tooltip, not jump-to-footnotes.

Rendered with custom override

renderer: 2 passes (parse refs + resolve map) · prompt complexity: medium (LLM must keep map consistent) · LLM stability: medium (map drift on regen)
C — JSON sidecar (positions)
Raw source from LLM (prose only)
EPS TTM is $3.18, beating consensus by $0.18. FY estimate revised up to $3.62; analyst price target consensus moved to $920.
Sidecar (separate file / column)
{ "citations": [ { "start": 14, "end": 19, "fact_id": "f_eps_aapl_q1_2026" }, { "start": 48, "end": 53, "fact_id": "f_eps_surprise_q1_2026" }, { "start": 81, "end": 86, "fact_id": "f_fyest_aapl_2026" }, { "start": 132, "end": 136, "fact_id": "f_target_aapl_2026_05" } ] }
Rendered (after position-based overlay)
EPS TTM is $3.18eps_q1, beating consensus by $0.18surprise. FY estimate revised up to $3.62fy_est; analyst price target consensus moved to $920target.
Fragile. If the LLM rewrites by 1 char on regen ("$3.18" → "$3.180"), all subsequent offsets shift; sidecar must be regenerated atomically. No overlay survives partial edits.
renderer: position overlay walk · prompt complexity: high (LLM must emit prose+sidecar atomically) · LLM stability: low (offset drift kills citations)

My read

C is fragile (offset drift on regen kills citations); we'd be re-deriving the sidecar every time the LLM touches the prose, which makes B+C indistinguishable but with extra bookkeeping. B's "standard markdown" advantage evaporates because the native footnote render is the wrong UX (superscript jump-to-bottom) — we'd override the renderer anyway, at which point the "standardness" gives us nothing. A is the simplest: marker travels with prose, single source of truth, one regex parse. The "raw markdown is noisier" cost is real but minor — users read rendered briefs, not raw. If we ever export raw .md (e.g. for pasting into Obsidian / Logseq), the markers degrade gracefully into visible text — no broken state.

→ Recommendation: A — [fact:<id>] inline

LLM is given a JSON list of facts in the prompt with their IDs; instructed to cite inline using [fact:<id>] with the exact ID. Renderer parses with /\[fact:([^\]]+)\]/g and substitutes CitationLink components resolving against the brief's facts payload (server-supplied alongside the prose, no offset bookkeeping). Hindsight recall toggle (FR81) drops or replaces markers post-dating the as_of timestamp — same regex pass with a filter. PDF export reuses the same parser. Implementation lands in slice R5 (research-brief-synthesis): prompt template + renderer in apps/web/src/lib/research/render-brief.ts.
