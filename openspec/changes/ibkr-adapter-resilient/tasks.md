## 6. Resilient reconnect loop with 5-attempt ceiling → `KillSwitchTripped`

- [x] 6.1 Implement `async def _resilient_reconnect_loop(self) -> None` per design D3: walk `attempt in range(5)`, `await asyncio.sleep(backoff_seconds(attempt, with_jitter=True))`, attempt `_connect_client` + `_send_heartbeat`, on success emit `broker.connection.restored` + run `_post_reconnect_reconciliation` + restart heartbeat task + return; on failure emit `BrokerConnectionAttemptFailed` + continue. After 5 exhausted attempts, publish `KillSwitchTripped(tenant_id=..., reason="ibkr.adapter.reconnect_exhausted", metadata={"attempts": 5, "backoff_sequence": [3, 6, 12, 24, 48]})` to the bus.
- [x] 6.2 Implement `async def _connect_client(self) -> None` — internal helper that constructs `self._client = self._client_factory()` + calls `connectAsync`. Catches auth-related errors (`IB.connectAsync` raises specific exception types per `ib_async`); on auth error, raises `IntegrationError("ibkr.auth.failed")` with NO retry (per design Non-Goal). Network errors propagate to the caller for retry.
- [x] 6.3 Add explicit non-retry path: if `_connect_client` raises `IntegrationError` with `type_uri` matching auth-failed pattern, `_resilient_reconnect_loop` immediately publishes `KillSwitchTripped(reason="ibkr.adapter.auth_failed")` and returns without exhausting attempts.
- [x] 6.4 Add unit test `apps/api/tests/unit/contexts/trading/brokers/test_resilient_reconnect.py` covering: 5 attempts → publishes KillSwitchTripped; success on attempt 3 → restored event + reconciliation runs; auth failure → immediate KillSwitchTripped without retry.
