## 4. sma_cross.py + unit tests

- [ ] 4.1 Create `apps/api/src/iguanatrader/contexts/trading/strategies/sma_cross.py` with `SMACrossStrategy(Strategy)` per design D3. `name() = "sma_cross"`, `version() = "1.0.0"`. Defaults: `fast=50, slow=200, vol_window=20, risk_pct=0.01, k=1.0`. `_validate_history` requires `len(history.bars) >= max(slow, vol_window)`. Entry condition: **golden cross** — `prev_fast <= prev_slow AND curr_fast > curr_slow` (cross just turned positive); pure `fast > slow` would over-emit on every bar. Volatility = numpy std-dev of returns over `vol_window`. Sizing: `quantity = risk_pct * equity / (k * volatility * entry)`.
- [ ] 4.2 Implement `_compute_smas(closes, fast, slow)` static helper returning `(fast_sma_series, slow_sma_series)` numpy arrays.
- [ ] 4.3 Create `apps/api/tests/unit/contexts/trading/strategies/test_sma_cross.py`: hand-crafted sequences. Test cases: (a) **insufficient warmup** (199 bars when slow=200) returns None; (b) **golden cross** (prev fast=99 < prev slow=100; curr fast=101 > curr slow=100) returns Proposal; (c) **already-crossed** (prev and curr both fast > slow) returns None — only emits on the cross itself; (d) **death cross** (fast crosses below slow) returns None or SELL signal per contract; v0 returns None to stay long-only; (e) **zero volatility** (constant prices) returns None defensively (would otherwise divide by zero); (f) **Decimal precision** at all quantity arithmetic; (g) **reasoning dict** contains `signal_source`, `sizing_rationale`, `stop_placement` keys.
- [ ] 4.4 Decision: SMA-cross has no natural stop (it's a trend-follow strategy). v0 sets `stop_price = entry * (Decimal("1") - 2 * volatility_pct)` where `volatility_pct = volatility / entry`; documented as "2 standard deviations below entry" in `reasoning.stop_placement`. Post-MVP can introduce a proper exit signal.
